# Database Systems & Architecture: Assignment 01

## Overview
This repository contains the complete execution of Module 2: SQL and Database Architecture. It demonstrates a comprehensive understanding of modern data engineering across multiple database paradigms, from traditional relational models to modern AI-driven vector storage.

## Project Structure
* **Part 1 (RDBMS):** Normalization of flat files to 3NF and complex SQL querying using MySQL/PostgreSQL syntax.
* **Part 2 (NoSQL):** JSON document design and query execution using MongoDB for flexible e-commerce cataloging.
* **Part 3 (Data Warehouse):** ETL data cleaning and Star Schema design (Fact and Dimension tables) for business intelligence reporting.
* **Part 4 (Vector Databases):** Generation of AI semantic embeddings using Python (`sentence-transformers`) and cosine similarity matrix visualization.
* **Part 5 (Data Lakes):** Cross-format data querying (CSV, JSON, Parquet) directly from raw storage using DuckDB.
* **Part 6 (Capstone Architecture):** End-to-end polyglot persistence system design for a hospital network, delineating strict OLTP and OLAP boundaries.

## Datasets
All raw data files (`orders_flat.csv`, `retail_transactions.csv`, `customers.csv`, `orders.json`, `products.parquet`) are stored locally in the `/datasets` directory.
